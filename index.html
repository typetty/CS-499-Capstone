Software Design and Engineering

Narrative (See enhanced artifacts here)

Briefly describe the artifact. What is it? When was it created?

The artifact that I chose to enhance for my “Software Design and Engineering” showcase was from the CS-360 class about mobile development. It is a mobile app that allows users to create events and track events in their lives. However, there were a few things I saw in the MainActivity.java file, which is the first screen and login screen, that I knew I could improve.

Justify the inclusion of the artifact in your ePortfolio. Why did you select this item? What specific components of the artifact showcase your skills and abilities in software development? How was the artifact improved?

I selected this item because I thought it was a perfect chance to showcase software design and engineering skills. The MainActivity class had all the UI and business logic contained within the same file. This is not the best practice, and I knew that I could enhance it to split the business logic into its own service class. This shows planning and thoughtfulness as you need to have as a software engineer. It also makes the code more modular and useful in other parts of the UI.

For example, to increase the complexity of this UserService that I added, I also included the ability to update the user’s username and password. I created a new UI page and class for a profile page. This also has access to the UserService to call functions to update the password or username. This is normal practice and good functionality for any modern application. Users want the ability to edit their information anytime, like their email address or other data. Now when the user needs to deal with anything on the user's object in the app, we can access it on the UI classes through the UserService. This also included a few new functions in the SQLiteManager class so that we can update the user information on the database.

Did you meet the course outcomes you planned to meet with this enhancement in Module One? Do you have any updates to your outcome-coverage plans?

I met the course outcomes and have completed everything I said I would. I separated the business logic from the login UI logic, I added methods in the UserService and the SQLiteManager, and I even added a UI page for editing the user profile data. This has made it completely functional and testable right now. My changes have made this a collaborative, organized environment for new developers to enter, excellent communication through a code review and new comments, and shows mastery of implementing solutions to meet needs and best practices.

Reflect on the process of enhancing and modifying the artifact. What did you learn as you were creating it and improving it? What challenges did you face?

One thing I learned was that it is hard to redesign the code after you have already written it. I tried to separate the business logic and UI logic on the MainActivity, and at first, it was hard to know where to start. However, knowing what I know now, I would definitely plan and engineer the software before implementing it. This would make mine or a developer's life much easier. Algorithms and Data Structures

Narrative (See enhanced artifacts here)

Briefly describe the artifact. What is it? When was it created?

The artifact that I chose to enhance for my “Software Design and Engineering” showcase was from the CS-360 class about mobile development. It is a mobile app that allows users to create events and track events in their lives. However, there were a few things I saw in the Event.java, EventAdapter.java, and EventListActivity.java files that I knew I could improve. This pertains particularly to my use of algorithms.

Justify the inclusion of the artifact in your ePortfolio. Why did you select this item? What specific components of the artifact showcase your skills and abilities in algorithms and data structure? How was the artifact improved?

I chose these artifacts because I believe they are perfect for showcasing my skills with algorithms and data structures. The first thing that I noticed and wanted to fix was that the method getEventPerId in the Event.java file was searching through every single event to find the correct and matching event. It was fine, but if we ever wanted to scale up the application, this was quickly going to become inefficient. What I implemented was adding each of the event objects into a hash map so that I could simply get the correct event by giving it an id. Now it is very efficient and gets the right record fast.

The second thing I wanted to change was to add a search capability. This included adding some UI elements to the list page as well as a search algorithm. I had to add a few methods. I added the updateEvents to the EventAdapter.java file to update the list and the events that appear filtered on the search. I also added the searchEventList method on EventListActivity.java to return and call the updateEvents method to show the correct events on the screen. Lastly, I added the searchEvents method to the Event.java file to implement the algorithm for searching. It takes the search input string and splits it into keywords to find. Then it parses through all the event names and checks if it contains the keywords, starts with them, matches the keyword exactly, or contains the keyword exactly. I made a hash map to contain scores or points to prioritize the results. That way, the most relevant results will show up on the top. Check out my Event.java file to see my implementation.

Did you meet the course outcomes you planned to meet with this enhancement in Module One? Do you have any updates to your outcome-coverage plans?

The outcome that I planned on meeting with this milestone was “design and evaluate computing solutions that solve a given problem using algorithmic principles and computer science practices and standards appropriate to its solution while managing the trade-offs involved in design choices.” Although the application was working and simpler before, we needed to improve it to allow for scalability. The problem we were facing was how we could search and select an event with many events in the system. Now, the code may be more complex (possibly harder for other developers to learn), but the use is much better. It accomplishes what we need in an efficient way.

Reflect on the process of enhancing and modifying the artifact. What did you learn as you were creating it and improving it? What challenges did you face?

I learned how much time and effort it takes to create a useful algorithm. It seems like you would just have some in your back pocket, but if you have never done what you are trying to accomplish before, it can be difficult. I had to look up ways others had implemented similar designs. It helped inform me of options I had to use. Getting a second opinion, I have learned, is very helpful when improving your code, especially algorithms.

Databases

Narrative (See enhanced artifacts here)

Briefly describe the artifact. What is it? When was it created?

The artifact that I chose to enhance for my “Software Design and Engineering” showcase was from the CS-360 class about mobile development. It is a mobile app that allows users to create events and track events in their lives. However, there were a few things I saw in the Event.java and SQLiteManager.java files that I knew I could improve. This pertains particularly to my configurations of the database.

Justify the inclusion of the artifact in your ePortfolio. Why did you select this item? What specific components of the artifact showcase your skills and abilities in software development? How was the artifact improved?

I chose these artifacts for a few reasons. The first was because I wasn’t hashing or encrypting my passwords anywhere. They were being stored as plain strings in the database, and it really wasn’t best practice. What I did was add a hashing function to take the incoming password string and hash it with SHA-256 encryption. I was thinking of different encryption types, but this seemed to fit it the best as I didn’t need to decrypt it once it was in the database.

The second thing that I wanted to fix was the event list being populated by getting all event records for all users. This is bad practice especially if we want to scale the app. It is important for security as well that you only return the records pertinent to the user. Changed the database creation methods to include a foreign key on the user id field on the event table and to enforce it. This way, whenever we get events back, it will have to match. Similarly, I added the user id as a parameter for the PopulateEventListArray method so that I could query and return the correct records to display. This allows the data to be filtered at the database level before it even gets to the app and user.

Did you meet the course outcomes you planned to meet with this enhancement in Module One? Do you have any updates to your outcome-coverage plans?

The outcomes that I tried to meet on this milestone were, “design and evaluate computing solutions that solve a given problem using algorithmic principles and computer science practices and standards appropriate to its solution while managing the trade-offs involved in design choices” and “develop a security mindset that anticipates adversarial exploits in software architecture and designs to expose potential vulnerabilities, mitigate design flaws, and ensure privacy and enhanced security of data and resources.” It shows good design while handling trade-offs because it causes more work for us right now, but in the long run, it will make the app more scalable and readable to other developers. Additionally, it really hits the security mindset outcome on the head because it fixes encryption issues with passwords not being hashed and users gaining access to all user records.

Reflect on the process of enhancing and modifying the artifact. What did you learn as you were creating it and improving it? What challenges did you face?

This enhancement was definitely harder for me than the others. The biggest problem was finding all the places that I needed to change in the code to match the new functions parameters and outputs. I would run the app and then have to check it again to fix another issue. It was a cycle that seemed to never end. But I see that fighting it until the end is worth it once the product does exactly what it needs to do.
